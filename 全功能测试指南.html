📋 识字训练应用 - 全功能测试指南

第一步：测试字库浏览

操作：打开应用后，默认显示"字库浏览"标签

  • ✅ 应该看到汉字网格（一、乙、上、下、口、目…）
  • ✅ 顶部显示：字库：XXX字 | 已掌握：0字 | 完成率：0%
  • ✅ 点击搜索按钮应该能过滤汉字

通过标准：看到至少30个汉字整齐排列

第二步：测试汉字详情

操作：点击任意一个汉字（如"跑"）

  • ✅ 弹出详情卡片，显示：大字、拼音、释义、年级
  • ✅ 显示笔画顺序（SVG动画）
  • ✅ 显示部首和笔画数
  • ✅ 底部有3个按钮：已掌握 / 加入错题本 / 识字自测

通过标准：能完整查看汉字详细信息

第三步：测试标记功能

操作：在详情卡片中点击"已掌握"

  • ✅ 按钮变绿，显示"已掌握 ✓"
  • ✅ 再次点击变回"标记已掌握"
  • ✅ 点击"加入错题本"，按钮变红

通过标准：按钮状态能切换，localStorage保存成功

第四步：测试错题本（关键）

操作：

  1. 在详情中点击"加入错题本"（至少加入2个字）
  2. 点击顶部"错题本"标签
    • ✅ 看到错题列表，显示加入的生字
    • ✅ 每个生字显示：字、拼音、错误次数、"移除"按钮
    • ✅ 底部有3个按钮："错题专练" / "造句专练" / "清空错题"

    通过标准：错题列表显示正确，有至少2个生字

🔥 第五步：测试造句专练（核心功能）

操作：在错题本页面，点击"造句专练"按钮

  • 预期1：页面下方展开造句区域（原来display:none）
  • 预期2：显示进度条（1 / 5）
  • 预期3：显示场景描述（如"在小区花园里"）
  • 预期4：显示2张生字卡片（emoji + 大字 + 拼音 + 释义）
  • 预期5：显示造句（如"我在公园里跑步，还要吃饭。"）
  • 预期6：句子中的生字有彩色高亮
  • 预期7：底部有"学会了！"按钮（带⭐图标）
  • 预期8：有"← 上一题"和"下一题 →"按钮

常见错误：

  • ❌ 点击"造句专练"没反应 → JavaScript报错
  • ❌ 显示空白 → 函数未定义或参数错误
  • ❌ 句子包含emoji（如"跑步🏃"）→ 模板未更新

通过标准：看到完整的造句卡片，句子通顺自然

第六步：测试星星奖励

操作：在造句卡片中点击"学会了！"按钮

  • ✅ 全屏弹出⭐大星星动画（1秒后消失）
  • ✅ 按钮文字变为"学会了！(1)"
  • ✅ 按钮变灰不可点击
  • ✅ 1秒后自动跳到下一题

通过标准：星星动画正常显示

第七步：测试翻页功能

操作：点击"下一题 →"按钮

  • ✅ 进度条更新（2 / 5）
  • ✅ 显示新的句子（不同生字组合）
  • ✅ 生字卡片更新
  • ✅ 点击"上一题"能回退

通过标准：5句话都能正常显示，不重复

第八步：测试"再来一次"

操作：完成5句话后，或点击"🎲 再来一次"按钮

  • ✅ 清空旧数据，生成新的5句话
  • ✅ 进度条重置为（1 / 5）
  • ✅ 星星积分重置为0

通过标准：新句子与之前不重复

第九步：测试识字自测

操作：点击"识字自测"标签 → 选择年级 → 点击"开始自测"

  • ✅ 显示选择题（选正确意思/读音）
  • ✅ 点击答案后显示对错（绿色/红色）
  • ✅ 点击"下一题"继续

通过标准：能正常答题并判断

第十步：测试阅读练习

操作：点击"阅读练习"标签 → 选择短文

  • ✅ 显示短文内容
  • ✅ 学过的字有颜色标记

通过标准：能正常显示短文

第十一步：测试学习统计

操作：点击"学习统计"标签

  • ✅ 显示4个指标：总字库、已掌握、错题数、完成率
  • ✅ 显示进度条
  • ✅ 显示各年级进度（1-6年级）

通过标准：统计数据与实际一致

✅ 全部测试完成！

如果所有步骤都通过，说明应用功能正常。

重点关注：第五步（造句专练）是本次优化的核心功能。

🐛 如果造句专练不工作，排查步骤：

  1. 按F12打开浏览器控制台
  2. 点击"造句专练"按钮
  3. 查看控制台是否有红色报错
  4. 常见报错：
    • Uncaught ReferenceError: getCommonWord is not defined → 函数缺失
    • Uncaught TypeError: Cannot read properties of undefined → 参数错误
    • 至少需要2个错题才能生成造句练习！ → 错题本为空
